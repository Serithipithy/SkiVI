  1. Introduction

    1. Purpose

      The main purpose of the project is to develop an application for learning different, unique and helpful skills and to make it more easier for the users to keep track of their leaning jurney.

    2. Document Conventions

      The documentation is made under the Scholarly convention and IEEE-Template.

    3. Intended Audience and Reading Suggestions

      The intended audience is represented by any person that wants to expand their knowledge with something new and unconventional or anyone that wants to contribute and make the application more accesible and wants to offer more information and courses.

      This document contains an overview of the application in order to provide information for all the users.

      The usual user will read some of the Overall Description, System Features and the rest of the audience that wants to provide something to the platform will read Overall Description, External Interface Requirements, System Features, Other Nonfunctional Requirements,Other Requirements Appendix A, B, C.

    4. Product Scope

      The scope of the product is to provide courses for a number of skills such as first aid, survival techniques, learn an instrument(theremin) or dead languages etc. Along with this, the application will provide a pleasant experience while using it so the user will learn at its pace.

  2. Overall Description

    1. Product Perspective

      SkiVi is a modular web application which acts like a virtual instructor and provides different sets of activities and resources for learning different skills such as: learn an instrument,survival tehniques,first aid,origami.

    2. Product Functions

      This application provides for the user the posibility to create an account so he/she can create his/her own personal learning plan.

      The user can check it's progress and for each course, he/she can start learning from the last watched material.

      Each course will be updated both automaticaly and at user demand, each update will be anounced on the course page.

    3. User Classes and Characteristics

      Typical users who want to use SkiVi to learn a new skill. Advanced users who want to perfect one of their skills.

    4. Operating Environment

      This Web aplication is compatible with any browser, it's interface provides support for big screens such as PC and mobile screens based on their dimensions, providing the users with a nice experience on their PC and on their mobile phone.

      The front-end of the application is developed with HTML and CSS.

  5. Other Nonfunctional Requirements

    1. Performance Requirements

      There is no special performance requirements.

    2. Safety Requirements

      Some of the courses may require special resources that can harm the person that uses them and they need to be cautious. In case of emergency they should call special care.

    3. Security Requirements

      After the user creates his/her account, their data will be secure without sharing it to a third party.

      For creating an account, the user will provide a username, his/her e-mail and a password.

      Every course will give credits to the creator and contact information of the course owner.

    4. Software Quality Attributes

      Due to its well designed and easy to use interface it can be used by both experts and typical users.

    5. Business Rules

      Contact information are available in order to communicate with the experts that want to provide courses for the application.
